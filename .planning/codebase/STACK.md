## Frontend

- **Next.js** `^16.2.3` — React framework with App Router
- **React** `^18.3.1` — UI library
- **TypeScript** `^5` — Type-safe JavaScript

### Build & Development
- **Package Manager:** yarn `1.22.22`
- **Bundler:** Turbopack (via Next.js)
- **Output:** Standalone mode for production
- **Dev Server:** Port 3000

### Styling
- **Tailwind CSS** `^4` — Utility-first CSS
- **Tailwind plugins:**

  - `tw-animate-css` — Additional animations
- **CSS Variables:** HSL-based theming (`--background`, `--foreground`, etc.)
- **PostCSS:** `@tailwindcss/postcss`

## Environment Configuration

### Frontend
- `NEXT_PUBLIC_TTS_API_BASE_URL` — TTS backend URL (default: `http://localhost:8000`)

### Backend
- `TTS_SERVER_RELOAD` — Enable auto-reload
- `HF_TOKEN` — Hugging Face token (optional)
- `SUPABASE_ACCESS_TOKEN` — For schema push (if used)

### Storage
- Voice files: `tts-server/voices/`
- Job temp files: `tts-server/tmp-jobs/`
- Project storage: Within `tmp-jobs/` via `ProjectStore`

## Dependencies to Note

### Frontend
- Heavy Radix UI dependency tree (25+ primitives)
- Framer Motion for animations
- MDX Editor for rich text

### Backend
- OmniVoice runtime (large models from Hugging Face)
- FFmpeg required for ASR fallback
- GPU optional but recommended
