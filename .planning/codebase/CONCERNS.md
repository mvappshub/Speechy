## Known Issues

### Backend Dependencies

**FFmpeg on Windows**
- **Issue:** `torchcodec` requires FFmpeg DLLs from "full-shared" build
- **Impact:** ASR fallback fails without proper FFmpeg setup
- **Workaround:** Use transcript sidecar files (`.txt` alongside `.wav`)
- **Status:** Documented in README

**GPU Memory**
- **Issue:** OmniVoice models require significant GPU memory
- **Impact:** May fail on consumer GPUs
- **Mitigation:** GPU detection in `infrastructure/gpu.py`

### Concurrency

**Job Semaphore**
- **Current:** Max 2 concurrent render jobs
- **Concern:** Hardcoded limit may not suit all deployments
- **Location:** `job_service.py` constructor

## Performance Concerns

### Frontend
- **Bundle Size:** Large Radix UI tree (25+ components)
- **MDX Editor:** Heavy dependency, may impact initial load

### Backend
- **Model Loading:** First request triggers model download
- **Cold Start:** Significant delay on first TTS request
- **Memory:** OmniVoice models resident in GPU memory

### Storage
- **tmp-jobs/** grows indefinitely without cleanup
- **TTL:** 15 minute default, but files may accumulate
