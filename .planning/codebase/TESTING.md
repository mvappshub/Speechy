# Testing

**Document:** TESTING.md  
**Project:** Předčítač Českého Textu (Czech Text Reader)  
**Last Updated:** April 2026

---

## Testing Philosophy

- Tests for domain logic and critical paths
- Test files co-located with source (frontend) or in `tests/` (backend)
- Minimal but meaningful coverage

---

## Frontend Testing

### Framework
- **Runner:** Node.js built-in test runner
- **Loader:** `tsx` for TypeScript
- **Assertion:** `node:assert` (strict mode)

### Test Command
```bash
npm run test:frontend
```

### Test File Pattern
```
src/
└── features/reader/
    ├── domain/
    │   ├── chunkSelection.test.ts
    │   ├── playback.test.ts
    │   └── textCleaning.test.ts
    └── infrastructure/
        ├── audioPlayer.test.ts
        └── ttsApi.test.ts
```

### Test Structure
```typescript
import { test } from "node:test";
import { strict as assert } from "node:assert";
import { functionUnderTest } from "./module";

test("should do something specific", () => {
  const input = "test";
  const result = functionUnderTest(input);
  assert.equal(result, expected);
});
```

### Domain Tests (Examples)

**`chunkSelection.ts`:**
- Block index selection from time position
- Edge cases (first block, last block)

**`playback.ts`:**
- Playback state transitions
- Progress calculation

**`textCleaning.ts`:**
- Whitespace normalization
- Special character handling

### Infrastructure Tests

**`ttsApi.ts`:**
- API endpoint construction
- Error handling

**`audioPlayer.ts`:**
- Audio element management
- Event handling

## Backend Testing

### Framework
- **Runner:** Python `unittest`
- **Discovery:** Auto-discovery in `tests/` directory

### Test Command
```bash
npm run test:backend
# Or directly:
python -m unittest discover -s tts-server/tests -v
```

### Test Files
```
tts-server/tests/
├── test_chunking.py        # Text splitting logic
├── test_governance_guard.py # Governance rules
├── test_http_app.py        # HTTP endpoint tests
└── test_job_service.py     # Job orchestration tests
```

### Test Structure
```python
import unittest
from domain.text_chunking import split_text_into_chunks

class TestChunking(unittest.TestCase):
    def test_splits_text_into_chunks(self):
        text = "Long text here..."
        chunks = split_text_into_chunks(text, max_chars=100)
        self.assertEqual(len(chunks), expected_count)
```

### Key Test Areas

| Module | Coverage |
|--------|----------|
| `text_chunking.py` | Splitting logic, edge cases |
| `job_service.py` | Job lifecycle, project management |
| `http.py` | Endpoint responses, validation |
| Governance | Architecture rules |

## Running Tests

### All Tests
```bash
npm run test
```

### Individual Suites
```bash
# Frontend only
npm run test:frontend

# Backend only
npm run test:backend
```

### Verification Pipeline
```bash
npm run verify
```
Runs: architecture checks → governance checks → tests → lint → build

## Coverage Gaps

### Not Tested (Observed)
- UI components (no component tests observed)
- End-to-end flows
- OmniVoice integration (requires models)
- File upload flows
- Audio playback integration

### Testing Strategy
- Domain logic: Unit tested
- Controllers: Integration tested via manual verification
- UI: Manual browser testing
- Backend: Unit + some integration
